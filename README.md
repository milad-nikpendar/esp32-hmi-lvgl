### High‑Performance HMI Framework for ESP32 + LVGL + FreeRTOS

ESP32‑HMI‑LVGL is a lightweight, modular, and high‑performance HMI framework designed for ESP32 boards using **LVGL**, **FreeRTOS**, **PSRAM**, and optional **DMA acceleration**. It provides a complete display + touch + UI runtime environment optimized for **PlatformIO**, supporting both **parallel** and **SPI** displays, capacitive and resistive touch controllers, and UI files generated by **EEZ Studio**.

This framework is **NOT** an Arduino library and must **NOT** be installed in the Arduino libraries folder.  
Instead, place the `esp-hmi/` folder directly inside your PlatformIO project for full control over configuration, memory, and hardware behavior.

---

## ✨ Features
- 🖥️ Support for 2.4", 2.8", 3.5", and 4.3" TFT displays  
- ⚡ High‑speed parallel 8‑bit (LCD_CAM) rendering with DMA  
- 🎨 LVGL 9.x fully integrated with custom memory allocators  
- 🧠 PSRAM‑aware buffer allocation for large UIs  
- 👆 Capacitive touch (GT911) with multitouch support  
- 👉 Resistive touch (XPT2046)  
- ⌨️ Analog keypad input  
- 🧵 Dedicated FreeRTOS tasks for LVGL & TFT rendering  
- 📈 Real‑time FPS monitoring  
- 🧩 Clean integration with UI files generated by **EEZ Studio**  
- 🔧 Modular architecture—easy to extend and customize

## 🧩 What the User Must Do

### ✔️ 1. Use PlatformIO  
This framework depends on FreeRTOS tasks, PSRAM allocators, and custom LVGL integration.

### ✔️ 2. Place the `esp-hmi/` folder inside your project  
Do **NOT** install it as a library.

```cpp
#include "esp-hmi/lvgl_functions.h"
```

### ✔️ 3. Generate UI files using EEZ Studio  
Download:  
https://github.com/eez-open/studio

Place generated files inside:

```
/ui/
    ui.c
    ui.h
    vars.c
    vars.h
    actions.c
    actions.h
```

### ✔️ 4. Configure your hardware using macros  
All configuration is done inside:

```
esp-hmi/lvgl_functions.h
```

## 🧠 Memory Configuration

```cpp
#define USE_PSRAM
// #define USE_DMA
```

### USE_PSRAM  
Allocates LVGL buffers in PSRAM using:

```cpp
heap_caps_malloc(..., MALLOC_CAP_SPIRAM | MALLOC_CAP_8BIT)
```

### USE_DMA  
Enables DMA acceleration for SPI or parallel transfers.

# 🧪 Quick Start Example (Simple)

```cpp
#include "esp-hmi/lvgl_functions.h"

void setup() {
    Serial.begin(115200);                     // Start serial monitor
    hmi_touch_init();                         // Initialize touch controller
    hmi_lvgl_init();                          // Initialize LVGL + Display + RTOS tasks
}

void loop() {}
```

# 🧵 Advanced Example (LVGL Render Task Callback)

#include "esp-hmi/lvgl_functions.h"

# 👆 Advanced Example (GT911 I2C Touch Callback)

```cpp
#include "esp-hmi/lvgl_functions.h"

// This callback runs INSIDE the I2C touch task (every 5ms)
// Safe place for reading RTC, sensors, or other I2C devices
void my_touch_callback() {

    // Example: Multi-touch detection
    if (touch_data.count > 1) {                                   // Multi-touch check
        Serial.println("Multi-touch detected!");                  // Debug print
    }

    // Example: Read RTC (DS3231)
    // rtc.readTime();                                            // Safe: runs inside same I2C task

    // Example: Read another I2C sensor
    // sensor.update();                                           // Safe: no collision with GT911

    // Example: Custom logic
    // my_app_logic();                                            // Runs synchronized with I2C
}

void setup() {
    Serial.begin(115200);                                         // Start serial monitor

    hmi_touch_init(my_touch_callback);                            // Initialize GT911 + user callback
    hmi_lvgl_init();                                              // Initialize LVGL + Display + RTOS tasks
}

void loop() {}
```

## 📘 Full ESP32 Diagnostics via esp32_functions.h

```cpp
#include "esp-hmi/lvgl_functions.h"
#include "esp-hmi/esp32_functions.h"     // Include system diagnostic utilities

void setup() {
    Serial.begin(115200);                // Start serial monitor
    delay(300);

    Serial.println("=== ESP32 SYSTEM DIAGNOSTICS ===");

    chipInfo();                          // Print chip model, cores, revision, ID, SDK, RAM, PSRAM
    Serial.println();

    Serial.printf("MAC Address: %s\n", readMacAddress());   // Print WiFi MAC address
    Serial.println();

    displayFreeHeap();                   // Print heap usage
    displayFreePsram();                  // Print PSRAM usage
    displayFreeSpiffs();                 // Print SPIFFS usage
}

void loop() {}
```
